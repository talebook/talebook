# 数据库迁移工具使用说明

## 简介

`migrate_db.py` 是一个智能的数据库迁移工具，用于自动对比 `models.py` 中定义的模型结构和实际数据库表结构，并添加缺失的字段。

## 使用场景

- ✅ 添加了新的模型字段，需要同步到数据库
- ✅ 从其他项目移植了功能，模型定义已更新但数据库未同步
- ✅ 需要安全的数据库结构迁移（不影响现有数据）

## 使用方法

### 在 Docker 容器中运行

```bash
# 进入容器
docker exec -it <容器名> bash

# 运行迁移脚本
python /var/www/talebook/webserver/migrate_db.py
```

### 在本地开发环境运行

```bash
cd /var/www/talebook/
python webserver/migrate_db.py
```

## 功能特点

### 1. 自动检测缺失字段

脚本会自动对比 `models.py` 中的模型定义和数据库实际结构，检测出：
- 缺失的字段
- 字段类型不匹配（未来功能）

### 2. 安全迁移

- ✅ 只添加缺失字段，不删除现有字段
- ✅ 使用事务确保数据一致性
- ✅ 详细的日志输出，方便排查问题

### 3. 支持的数据类型

- INTEGER（整数）
- VARCHAR(n)（字符串）
- BOOLEAN（布尔值）
- DATETIME（日期时间）
- FLOAT（浮点数）
- TEXT（文本）

## 注意事项

1. **备份数据库**：虽然脚本很安全，但建议在运行前备份数据库
2. **只添加不删除**：脚本只会添加缺失字段，不会删除现有字段
3. **默认值**：如果字段有默认值，会自动应用到新字段
4. **NOT NULL 约束**：如果字段定义为 NOT NULL 且有默认值，会添加 NOT NULL 约束

## 与 --syncdb 的区别

| 特性 | migrate_db.py | --syncdb |
|------|---------------|----------|
| 添加新字段 | ✅ 支持 | ❌ 不支持（仅创建新表） |
| 删除字段 | ❌ 不支持 | ❌ 不支持 |
| 创建新表 | ❌ 不支持 | ✅ 支持 |
| 影响现有数据 | ❌ 不影响 | ❌ 不影响 |
| 适用场景 | 生产环境 | 开发环境/初次安装 |

## 故障排除

### 问题：提示"表不存在"

**原因**：模型中定义的表在数据库中不存在

**解决**：先运行 `python server.py --syncdb` 创建所有表，再运行迁移脚本

### 问题：提示"字段已存在"

**原因**：字段已经存在于数据库中

**解决**：这是正常现象，脚本会跳过已存在的字段

### 问题：迁移失败，提示语法错误

**原因**：SQLite 不支持某些复杂的 ALTER TABLE 操作

**解决**：查看错误日志，可能需要手动执行 SQL 命令

## 自动化集成

可以将此脚本集成到 CI/CD 流程中：

```bash
# 部署时自动运行迁移
python /var/www/talebook/webserver/migrate_db.py || exit 1

# 然后重启服务
systemctl restart talebook
```

## 技术实现

- 使用 SQLAlchemy 的 `inspect()` API 获取数据库元数据
- 使用 `Base.registry._class_registry` 获取所有模型定义
- 使用原生 SQL 的 `ALTER TABLE ... ADD COLUMN` 语句添加字段
- 使用事务确保操作的原子性
