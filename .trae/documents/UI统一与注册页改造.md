# UI统一与注册页改造计划

## 任务目标
为 `login.vue` 与 `signup.vue`（注册页）添加与 `index.vue` 同款的顶部栏、侧边栏组件，确保三个页面的导航、样式和交互体验完全统一。

## 实施步骤

### 1. 分析当前布局结构
- **现状**：`index.vue` 使用 `AppHeader.vue` 组件实现顶部栏和侧边栏
- **问题**：`login.vue` 和 `signup.vue` 使用简单的 `v-toolbar`，未使用统一的 `AppHeader.vue` 组件

### 2. 修改 `login.vue` 页面
- 移除当前的简单 `v-toolbar` 结构
- 引入 `AppHeader.vue` 组件
- 调整页面布局，确保登录表单在顶部栏和侧边栏下方正确显示
- 保持原有的登录和忘记密码功能逻辑不变

### 3. 修改 `signup.vue` 页面
- 移除当前的简单 `v-toolbar` 结构
- 引入 `AppHeader.vue` 组件
- 调整页面布局，确保注册表单在顶部栏和侧边栏下方正确显示
- 保持原有的注册功能逻辑不变

### 4. 验证和测试
- 确保三个页面的顶部栏和侧边栏样式一致
- 验证导航功能正常（侧边栏菜单、顶部栏搜索等）
- 确保登录和注册功能在新布局下正常工作
- 测试响应式布局，确保在不同设备上显示正常

## 技术要点
- 使用 Nuxt 3 的组件引入方式
- 确保页面内容区域正确嵌套在布局组件内
- 保持原有功能逻辑不变，仅修改布局结构
- 遵循项目现有的代码风格和命名规范
